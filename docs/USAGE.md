# Документация по использованию SQL AI Agent

## Примеры вопросов

- Топ-10 работодателей по количеству вакансий
- Покажи вакансии с зарплатой от 100 000 рублей
- Сколько вакансий в Москве?
- Вакансии по городам: название города и количество
- Средняя зарплата по типам занятости (employment_name)
- Последние 20 опубликованных вакансий

## Ограничения

- Агент генерирует только **SELECT** (при включённом `AGENT_READ_ONLY`, по умолчанию).
- Данные — только то, что уже загружено в БД скриптом `load_hh_vacancies.py` (по умолчанию 2 страницы по 50 вакансий по запросу «Python»). Для большего объёма измените параметры в скрипте.

## Troubleshooting

| Проблема | Решение |
|----------|--------|
| `OPENAI_API_KEY не задан` | Задайте переменную в `.env` или в окружении Docker. |
| Ошибка подключения к БД | Проверьте `DATABASE_URL`. В Docker используйте хост `postgres`, не `localhost`. |
| Пустая таблица результатов | Сначала выполните `python scripts/load_hh_vacancies.py` (или через `docker compose exec app ...`). |
| «Не удалось извлечь SQL» | Модель иногда отвечает текстом вместо блока SQL. Попробуйте переформулировать вопрос или другую модель. |
| **404 Route POST:/v1/chat/completions not found** | Текущий провайдер не поддерживает OpenAI-совместимый chat API. **api.cursor.com так не работает.** Укажите OpenAI (`OPENAI_BASE_URL=https://api.openai.com/v1` + ключ OpenAI) или OpenRouter (`OPENAI_BASE_URL=https://openrouter.ai/api/v1` + ключ с openrouter.ai/keys). |
